# 测试数据使用说明

## 概述

项目已配置完整的测试数据系统，可以在本地开发时使用模拟数据，而不影响生产环境。

## 快速开始

### 1. 启用测试数据

编辑 `services/apiService.ts` 文件，找到第14行：

```typescript
const USE_MOCK_DATA = !isProduction && true; // 修改这里的 true/false 来切换
```

- **启用测试数据**：设置为 `true`
- **使用真实接口**：设置为 `false`

### 2. 可用的测试数据

项目提供了两套完整的测试数据（位于 `services/mockData.ts`）：

#### mockUserData（完整数据）
- 姓名：张三
- 工号：20210001
- 岗位：高级软件工程师
- 特点：数据丰富，包含所有模块的完整信息

#### mockUserDataMinimal（精简数据）
- 姓名：李四
- 工号：20220015
- 岗位：产品经理
- 特点：数据较少，适合测试边界情况

### 3. 切换测试数据

编辑 `services/apiService.ts` 第31行，修改返回的数据：

```typescript
return {
  success: true,
  data: mockUserData  // 或改为 mockUserDataMinimal
};
```

## 环境说明

### 开发环境（Development）
- 可以使用测试数据或真实接口
- 通过 `USE_MOCK_DATA` 标志控制

### 生产环境（Production）
- **自动禁用**测试数据
- 始终使用真实 API 接口
- 无需任何配置更改

## 数据字段说明

测试数据包含完整的年度报告所需字段：

| 类别 | 字段示例 |
|------|---------|
| 基本信息 | 姓名、工号、岗位、毕业院校 |
| 在岗数据 | 在岗时长、登录次数、最早上线、最晚下线 |
| 效能数据 | 总操作数、已办数量、处理率 |
| 出差数据 | 出差次数、出差天数、出差城市 |
| 学习数据 | 考试次数、最高分、题目数 |
| 互动数据 | 阅读数、点赞数、附件数量 |
| 奖励数据 | 奖励数量、奖励列表 |

## 自定义测试数据

如需添加自己的测试数据，在 `services/mockData.ts` 中创建新的数据对象：

```typescript
export const myCustomData: AnnualReportData = {
  // 复制 mockUserData 的结构
  // 修改为你需要的值
  XingMing: "你的姓名",
  GongHao: "你的工号",
  // ... 其他字段
};
```

然后在 `apiService.ts` 中引用：

```typescript
import { myCustomData } from './mockData';

// 在 getUserSummary 方法中
return {
  success: true,
  data: myCustomData
};
```

## 注意事项

⚠️ **重要提示**：
1. 生产环境部署前，确认 `USE_MOCK_DATA` 的逻辑正确（已自动配置）
2. 测试数据仅在开发环境生效，生产环境会自动忽略
3. 模拟数据包含 800ms 的网络延迟，模拟真实请求体验

## 调试信息

启用测试数据后，控制台会显示：
```
🎭 [API Service] 使用模拟数据进行测试
```

这表示当前正在使用本地测试数据。
